# Bitrix Mapper — подготовка данных к импорту в Битрикс 24

На входе: JSON-файл (`input/leads.json`) — массив объектов с полями как в экспорте Битрикс (ключи совпадают с названиями колонок).

На выходе: CSV-файл (`output/leads_ready.csv`) с разделителем `;` и кодировкой UTF-8 с BOM (удобно открывать в Excel).

## Установка

Дополнительные зависимости не требуются — только **Python 3** (стандартная библиотека).

## Настройки

Разделитель **выходного** CSV задаётся в `config.py` (`CSV_OUTPUT_DELIMITER`), по умолчанию `;`.

При ошибках в отдельных записях программа завершится с **кодом 1**; CSV при этом всё равно перезаписывается (в конце файла — блок «Ошибки»).

## Ограничение

Обрабатываются только **стандартные поля** (колонки из шаблона, 64 заголовка). Пользовательские поля (UF_*, UTM и т.п.) в выходной CSV не попадают.

## Запуск

```bash
python3 main.py
```

Графический режим: `python3 app.py` (выбор JSON-файла, результат — тот же `output/leads_ready.csv`).

## Состав проекта

- `main.py` — консоль
- `app.py` — Tkinter
- `mapper.py` — чтение JSON, маппинг, CSV
- `config.py` — пути и константы
- `validators.py` — телефоны, email, даты, числа
